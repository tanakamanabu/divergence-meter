# 回路図入力ガイド（KiCad）

8桁 IN-14 ダイバージェンスメーターの回路図を、迷わず・手戻りなく入力するための手順とルール。

## 1. 全体方針：トップレベルシートで分ける

ブロックが多いので、1枚に詰め込まず**複数のトップレベルシート**（フラットな複数ページ）に分割する。
ページ間は**グローバルラベル**（同名＝同一ネット）で接続するので、階層ピンの定義は不要。

```
divergence-meter（ルート）
├ power      電源：5V入力 → 180V昇圧、3V3、GND配電
├ control    XIAO ESP32-C3 + 74HCT595×2 + RTC + ボタン
├ drivers    74141×2 + アノードHV-SW×4 + ドットドライバ×4
└ tubes      IN-14 ×8（カソード/アノード/ドットの接続）
```

各ページは `power.kicad_sch` のように別ファイルになる。

**作り方（KiCad 9/10）:**
1. 「回路図の階層」パネルを右クリック → **新しいトップレベルシート** → 名前 `power`
2. 同様に `control` / `drivers` / `tubes` を追加

> なぜ階層（入れ子）シートでなくトップレベルシートか：本ガイドのネット命名規約どおり
> **全ネットをグローバルラベルで命名**しているため、各ページに同名ラベルを置くだけで
> 横断接続される。フラットな方が navigate も楽。

**繰り返し回路（アノードHV-SW×4・ドットドライバ×4）の扱い:**
- 小さい回路なので **`drivers` ページ内でコピペ ×4** が一番早い（実用上これで十分）。
- 「1 枚を使い回したい」部分だけ、そのページ内に**階層サブシート**を入れて複数インスタンス化してもよい
  （トップレベルシート自体は使い回し不可）。

## 2. 入力する順番

電気の流れ＝下流が上流に依存するので、**上流（電源）から**入れると ERC が早く通る。

1. **電源スケルトン** — `+5V` `+180V` `+3V3` `GND` の電源シンボルと `PWR_FLAG` を先に置く
2. **昇圧回路**（power）— MAX1771, L, Q, D, Rsense, C_in, C_out, R1/R2, Rb
3. **制御**（control）— XIAO、74HCT595×2（パスコン 0.1µF 各）、RTC、ボタン
4. **カソードドライバ**（drivers）— 74141 ×2
5. **アノードHV-SW ×4 / ドットドライバ ×4**（drivers）
6. **IN-14 ×8**（tubes）— カソード/アノード/ドットの各バスへ接続
7. **コネクタ** — USB 5V 入力（XIAO 経由 or 別途）、外部端子

## 3. ネット命名規約

ラベル（グローバルラベル）で配線する。長い線を引かずラベルで繋ぐとシートをまたげる。

| 種別 | ネット名 | 備考 |
|---|---|---|
| 電源 | `+5V` `+180V` `+3V3` `GND` | `+180V` は HV ネットクラス |
| 昇圧制御 | `SW`（スイッチノード）, `FB` | |
| 595 制御 | `SR_DATA` `SR_CLK` `SR_LATCH` `SR_OE` | XIAO の MOSI/SCK/GPIO20/GPIO21 |
| 74141-A BCD | `A_BCD0..A_BCD3` | 595#1 Q0-Q3 |
| 74141-B BCD | `B_BCD0..B_BCD3` | 595#1 Q4-Q7 |
| アノード選択 | `SEL1..SEL4` | 595#2 Q0-Q3 → 各 HV-SW の制御入力 |
| アノードバス | `AN1..AN4` | HV-SW 出力 → 各列のアノード（HV） |
| カソード A | `KA0..KA9` | 74141-A 出力。バス表記 `KA[0..9]`（HV） |
| カソード B | `KB0..KB9` | 74141-B 出力。バス表記 `KB[0..9]`（HV） |
| ドット | `DOT_LA` `DOT_RA` `DOT_LB` `DOT_RB` | 595#2 Q4-Q7 → ドットドライバ（HV） |
| I2C | `SDA` `SCL` | RTC |

- カソードは**バス表記**（`KA[0..9]`）＋バスエントリで束ねると見やすい。
- リファレンス：IC=`U`、トランジスタ=`Q`、受動=`R/C/L/D`、コネクタ=`J`、**ニキシー管=`V1..V8`**。

## 4. ネットクラスで HV を分離

Schematic Setup → Net Classes に **`HV`** を追加し、`+180V` / `AN*` / `KA*` / `KB*` / `DOT_*` を割り当てる。
PCB 段階でこのクラスにだけ**太い配線幅・広い沿面距離（クリアランス）**を設定できる。

## 5. シンボル準備（IN-14 は自作）

`lib/custom.kicad_sym` に IN-14 シンボルを作る。ピンはデータシート通り：

| Pin | 名称 | Pin | 名称 |
|---|---|---|---|
| 1 | Anode | 8 | K6 |
| 2 | LHDP | 9 | K7 |
| 3 | K1 | 10 | K8 |
| 4 | K2 | 11 | K9 |
| 5 | K3 | 12 | K0 |
| 6 | K4 | 13 | RHDP |
| 7 | K5 | | |

- 電気タイプ：Anode = `Passive`、各カソード/ドット = `Passive`（能動判定不要）。
- 他は標準ライブラリ or SnapEDA（74HCT595, 74141/K155ID1, MAX1771, MPSA42/92）。
- XIAO は Seeed 提供の KiCad シンボル/フットプリント（[LIBRARIES.md](../LIBRARIES.md) 参照）。

## 6. KiCad 操作の勘所

- **PWR_FLAG**：電源を「供給」するネット（コネクタの `+5V`、昇圧出力の `+180V` など）に必ず付ける。
  付けないと ERC が「電源ネットにドライバが無い」と誤検出する。
- **グローバルラベル**で配線（シート跨ぎ）。同名ラベル＝同一ネット。
- **0.1µF パスコン**を各 IC（XIAO 除く各 595・74141）の電源ピン直近に。
- **アノード抵抗 `Ra` は各管に**（8 本）。ドットは `R_dot`（直列）で電流制限。
- 入れ終わるごとに **ERC を実行**（早期に矛盾を潰す）。
- 記号を置いたら **Tools → Annotate**、続いて **Assign Footprints**。

## 7. PCB へ渡す前のチェック

- [ ] ERC がクリーン（HV ネットの孤立ピン無し）
- [ ] すべてのシンボルにフットプリント割り当て済み
- [ ] `HV` ネットクラスに `+180V`/`AN*`/`KA*`/`KB*`/`DOT_*` が入っている
- [ ] ブリーダ `Rb` が `+180V`–`GND` 間に入っている（安全）
- [ ] 各 IC にパスコン
- 完了したら **Update PCB from Schematic** で基板へ。
